# Frontend Design Summary

Design guidelines for enhancing the distinctiveness and aesthetic quality of UI.
Apply only when explicitly requested.

## Design Thinking (decide first)
- Purpose: What problem is this UI solving, and for whom?
- Tone: Choose one bold direction (e.g. minimal / brutalist / editorial / playful)
- Constraints: Technical constraints and a11y
- Differentiation: Decide the one "memorable point" of this UI

## Implementation Quality Standards
- Production-grade and actually functional
- Visually strong impression with a consistent direction
- Attention to detail (spacing, hierarchy, states)

## Design Guidelines
- Typography: Pair an eye-catching display font with a readable body font
- Color: Clearly define a primary color and a sharp accent
- Motion: Focus on one large, singular effect (avoid scattering)
- Space: Deliberately choose between generous whitespace or high density
- Background: Atmospheric layering or texture (if needed)

## Notes
- The strength of intent is what matters in direction — not how flashy it is.
- Match the complexity of implementation to the design direction.
